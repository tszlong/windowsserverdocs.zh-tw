# <a name="plan-a-software-defined-network-infrastructure"></a>規劃軟體定義網路的基礎結構

>適用於：Windows Server (半年度管道)、Windows Server 2016

瞭解軟體定義網路基礎結構的部署規劃，包括硬體和軟體必要條件。


## <a name="prerequisites"></a>必要條件
本主題描述數個硬體和軟體必要條件，包括：

-   **已設定的安全性群組、記錄檔位置和動態 DNS 註冊**您必須準備資料中心以進行網路控制站部署，這需要一或多部電腦或 vm，以及一部電腦或 VM。 在您可以部署網路控制站之前，您必須先設定安全性群組、記錄檔位置 (視需要) 和動態 DNS 註冊。  如果您尚未準備您的資料中心進行網路控制站部署，請參閱[部署網路控制卡的安裝和準備需求](Installation-and-Preparation-Requirements-for-Deploying-Network-Controller.md)以取得詳細資料。

-   **實體網路** 您需要存取您的實體網路裝置，以設定 Vlan、路由、BGP、資料中心橋接 (ETS) 如果使用 RDMA 技術，而且資料中心橋接 (PFC) （如果使用以 RoCE 為基礎的 RDMA 技術）。 本主題顯示手動交換器設定，以及第3層交換器/路由器上的 BGP 對等互連，或 (RRAS) 虛擬機器的路由及遠端存取服務器。

-   **實體計算主機** 這些主機會執行 Hyper-v，而且必須裝載 SDN 基礎結構和租使用者虛擬機器。  為了達到最佳效能，這些主機需要特定的網路硬體，稍後會在 [**網路硬體**] 區段中加以說明。


## <a name="physical-network-and-compute-host-configuration"></a>實體網路和計算主機設定

每個實體計算主機都需要透過一或多個網路介面卡連接到實體交換器埠 (s) 的網路連線。  第2層[VLAN](https://en.wikipedia.org/wiki/Virtual_LAN)支援網路劃分成多個邏輯網路區段。

>[!TIP]
>在存取模式或未標記中，對邏輯網路使用 VLAN 0。

>[!IMPORTANT]
>Windows Server 2016 軟體定義的網路功能支援 underlay 和重迭的 IPv4 定址。 不支援 IPv6。

### <a name="logical-networks"></a>Logical networks

#### <a name="management-and-hnv-provider"></a>管理和 HNV 提供者

所有實體計算主機都必須存取管理邏輯網路和 HNV 提供者邏輯網路。  針對 IP 位址規劃用途，每個實體計算主機至少必須有一個從管理邏輯網路指派的 IP 位址。 網路控制卡需要保留的 IP 位址，才能做為 REST IP 位址。

DHCP 伺服器可以自動指派管理網路的 IP 位址，您也可以手動指派靜態 IP 位址。 SDN 堆疊會自動為 HNV 提供者邏輯網路指派 IP 位址，以便從指定的 IP 集區，以及由網路控制站管理的個別 Hyper-v 主機。

>只有在網路控制卡主機代理程式收到特定租使用者虛擬機器的網路原則之後，網路控制站才會將 HNV 提供者 IP 位址指派給實體計算主機。

## <a name="routing-infrastructure"></a>路由基礎結構

如果您要使用腳本部署 SDN 基礎結構，則必須在實體網路上將管理、HNV 提供者、傳輸和 VIP 子網彼此路由傳送。

\( \) 使用內部 BGP 對等互連，將 SLB/MUX 和 RAS 閘道通告到實體網路中的路由資訊，例如 VIP 子網的下一個躍點。 VIP 邏輯網路沒有指派 VLAN，而且在第2層交換器中未預先設定 (例如，機架頂端交換器) 。

您需要在您的 SDN 基礎結構所使用的路由器上建立 BGP 對等互連，以接收 SLB/Mux 和 RAS 閘道所公告之 VIP 邏輯網路的路由。 BGP 對等互連只需要以一種方式從 SLB/MUX 或 RAS 閘道 (至外部 BGP 對等) 進行。  在路由的第一層中，您可以使用靜態路由或其他動態路由通訊協定（例如 OSPF），如先前所述，VIP 邏輯網路的 IP 子網首碼必須可從實體網路路由傳送至外部 BGP 對等互連。

BGP 對等互連通常會在受管理交換器或路由器中設定為網路基礎結構的一部分。 您也可以在具有遠端存取服務器的 Windows 伺服器上設定 BGP 對等， (RAS) 角色以「僅限路由」模式安裝。 網路基礎結構中的此 BGP 路由器對等必須設定為具有自己的 ASN，並允許從指派給 SDN 元件 \( SLB/MUX 和 RAS 閘道的 ASN 進行對等互連 \) 。 您必須取得實體路由器的下列資訊，或從該路由器控制的網路系統管理員：

- 路由器 ASN
- 路由器 IP 位址
- SDN 元件所使用的 ASN (可以是私用 ASN 範圍內的任何數位) 

>[!NOTE]
>SLB/MUX 不支援四個位元組的 Asn。 您必須將兩個位元組 Asn 配置給 SLB/MUX 和它所連接的路由器 wo。 您可以在您的環境中的其他地方使用4個位元組的 Asn。

您或您的網路系統管理員必須設定 BGP 路由器對等，以接受來自您 RAS 閘道和 SLB/Mux 所使用之傳輸邏輯網路的 ASN 和 IP 位址或子網位址的連線。

如需詳細資訊，請參閱[邊界閘道協定 (BGP) ](../../../remote/remote-access/bgp/Border-Gateway-Protocol-BGP.md)。

## <a name="default-gateways"></a>預設閘道
設定為連線到多個網路的電腦，例如實體主機和閘道虛擬機器，必須只設定一個預設閘道。 在用來連線到網際網路的介面卡上設定預設閘道。

針對虛擬機器，請遵循下列規則來決定要使用哪一個網路做為預設閘道：

1. 如果虛擬機器連線到傳輸網路，或如果它是多路連接到傳輸網路或任何其他網路，請使用傳輸邏輯網路做為預設閘道。
2. 如果虛擬機器只連線到管理網路，請使用管理網路做為預設閘道。
3. 針對 SLB/Mux 和 RAS 閘道使用 HNV 提供者網路。 請勿使用 HNV 提供者網路做為預設閘道。
4. 請勿將虛擬機器直接連線到 Storage1、Storage2、Public VIP 或私人 VIP 網路。

針對 Hyper-v 主機和存放裝置節點，使用管理網路做為預設閘道。  存放裝置網路絕不能指派預設閘道。

### <a name="network-interface-cards-nics"></a>網路介面卡 (NIC)

Hyper-v 主機和存放主機中所使用的網路介面卡 (Nic) 需要特定功能才能達到最佳效能。

 (RDMA) 的遠端直接記憶體存取是一種核心略過技術，可讓您在不使用主機 CPU 的情況下傳輸大量資料，這會釋出 CPU 來執行其他工作。

交換器內嵌小組 (設定) 是替代的 NIC 小組解決方案，您可以在包含 Hyper-v 的環境和 Windows Server 2016 中的軟體定義網路 (SDN) 堆疊中使用。 將一些 NIC 小組功能整合到 Hyper-v 虛擬交換器。

如需詳細資訊，請參閱[ (RDMA 的遠端直接記憶體存取) 和切換內嵌小組 (設定) ](../../../virtualization//hyper-v-virtual-switch/RDMA-and-Switch-Embedded-Teaming.md)。

為了考慮 VXLAN 或 NVGRE 封裝標頭所造成的租使用者虛擬網路流量中的額外負荷，第2層網狀架構網路 (交換器和主機) 的 MTU 必須設定為大於或等於1674個位元組， \( 包括第2層 Ethernet 標頭 \) 。

支援 new *EncapOverhead* advanced adapter 關鍵字的 nic 會透過網路控制站主機代理程式自動設定 MTU。 不支援 new *EncapOverhead*關鍵字的 nic 必須使用*JumboPacket* \( 或對等關鍵字，在每個實體主機上手動設定 MTU 大小 \) 。

|適用于 SLB/MUX 對等互連的 RAS 閘道 BGP 路由器<p> (也會使用 ToR 參數作為 BGP 路由器) |2個 vcpu|2 GB|適用于 OS 磁片磁碟機的 75 GB|


如果您使用 VMM 進行部署，VMM 和其他非 SDN 基礎結構需要額外的基礎結構虛擬機器資源。 如需其他資訊，請參閱[System Center Technical Preview 的最低硬體建議。](https://technet.microsoft.com/library/dn997303.aspx)

## <a name="extending-your-infrastructure"></a>擴充您的基礎結構
您的基礎結構的大小和資源需求取決於您打算裝載的租使用者工作負載虛擬機器。 基礎結構虛擬機器的 CPU、記憶體和磁片需求 (例如：網路控制站、SLB、閘道等 ) 列在上表中。 您可以視需要新增更多的基礎結構虛擬機器來相應放大。 不過，在 Hyper-v 主機上執行的任何租使用者虛擬機器都有自己的 CPU、記憶體和磁片需求，您必須加以考慮。

當租使用者工作負載虛擬機器在實體 Hyper-v 主機上開始耗用太多資源時，您可以藉由新增額外的實體主機來擴充您的基礎結構。 這可以透過 Virtual Machine Manager 或使用 PowerShell 腳本來完成， (視您最初部署基礎結構的方式而定) 藉由網路控制站建立新的伺服器資源。 如果您需要為 HNV 提供者網路新增額外的 IP 位址，您可以建立新的邏輯子網， (與主機可以使用的對應 IP 集區) 。
